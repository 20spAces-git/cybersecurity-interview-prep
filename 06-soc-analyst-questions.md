# Security Operations Center (SOC) Analyst Interview Questions

## What is a Security Operations Center (SOC)?

Answer:

A Security Operations Center (SOC) is a team responsible for continuously monitoring, detecting, investigating, and responding to cybersecurity threats.

SOC analysts work to identify suspicious activity before it becomes a major security incident.

---

## What Does a SOC Analyst Do?

Answer:

A SOC Analyst is responsible for:

- Monitoring security alerts
- Investigating suspicious activity
- Reviewing logs
- Escalating incidents
- Documenting findings
- Responding to cybersecurity threats

---

## What is a SIEM?

Answer:

A SIEM (Security Information and Event Management) system collects, correlates, and analyzes logs from multiple systems to help detect security threats.

Examples include:

- Microsoft Sentinel
- Splunk
- Google Chronicle
- QRadar

---

## What is an Alert?

Answer:

An alert is a notification generated by a security tool when suspicious or potentially malicious activity is detected.

Not every alert represents an actual attack, so analysts investigate to determine whether it is a true positive or false positive.

---

## What is a False Positive?

Answer:

A false positive occurs when a security tool incorrectly identifies legitimate activity as malicious.

Example:

An employee logs in from a new location while traveling, triggering an alert even though the activity is legitimate.

---

## What is a True Positive?

Answer:

A true positive is an alert that correctly identifies malicious or unauthorized activity.

Example:

A malware infection detected on a workstation.

---

## What Information Would You Review During an Investigation?

Answer:

I would review:

- User activity
- Authentication logs
- Network traffic
- Process activity
- System logs
- Endpoint alerts
- Email activity

---

## What Should You Do First When Receiving a Security Alert?

Answer:

1. Review the alert details.
2. Validate whether the activity is suspicious.
3. Gather additional evidence.
4. Determine the scope of the incident.
5. Escalate or contain the issue if necessary.
6. Document all findings.

---

## Why is Documentation Important in a SOC?

Answer:

Documentation creates a record of:

- Actions taken
- Evidence collected
- Investigation timeline
- Recommendations

Good documentation helps future investigations and supports communication between team members.

---

## What Skills Make a Good SOC Analyst?

Answer:

A successful SOC analyst should have:

- Critical thinking
- Attention to detail
- Communication skills
- Networking knowledge
- Linux fundamentals
- Windows fundamentals
- Log analysis
- Incident response knowledge
